GameCraft-Bench

GameCraft-Bench asks a coding agent to turn a product-style game brief into a complete, playable Godot 4 project. Each submission includes a game project and replayable input traces. The verifier launches the game, replays the traces, records gameplay videos, samples visual evidence, and scores the result with a task rubric.

Abstract

Game development is an important application for coding agents, where a user specifies an interactive experience and the agent must turn that intent into a working game artifact. This setting is challenging because game development happens inside an engine, where scripts, scene graphs, assets, input maps, rendering, visual feedback, and runtime configuration must work together as an executable interactive system, rather than merely producing compilable code.

Evaluating this setting requires three properties: P1: Engine-grounded Construction, P2: Artifact Completeness, and P3: Interactive Playability Evaluation. Existing game-generation benchmarks cover parts of this space but do not jointly satisfy these properties. GameCraft-Bench is an end-to-end benchmark for 2D game development in Godot, containing 140 tasks across 15 game families.

For each task, an agent receives a game brief, shared asset libraries, and a fixed workspace, then must produce a complete game project and demonstration traces. Evaluation of frontier coding agents shows that end-to-end game development remains highly challenging: even the strongest agent scores only 41.46%, and most agents score below 40.00%.

Evaluation Protocol

GameCraft-Bench evaluates complete playable artifacts. Each agent receives a game brief, shared asset libraries, and a fixed workspace, then submits a Godot project plus replayable demonstration traces.

  1. Build and launch the Godot project.
  2. Replay mouse and keyboard traces at 30 FPS.
  3. Record gameplay videos and sample visual evidence.
  4. Score rubric items with a multimodal judge.

Core Mechanics

Does the requested gameplay loop work under player input?

Content Depth

Is there enough progression, content, and state variation?

Functional Visuals

Can players read state, feedback, transitions, and outcomes?

Art and Presentation

Does the game feel visually coherent and appropriately styled?

Benchmark Properties

Engine-grounded Construction

Agents work in a real game engine with scenes, scripts, assets, input maps, and runtime configuration.

Artifact Completeness

The unit of evaluation is a complete playable vertical slice, not an isolated code fragment.

Interactive Playability Evaluation

Scores come from replayed player interaction and recorded gameplay evidence, not source code alone.

Key Findings

Mechanics are easier than complete games.

Top agents score higher on core mechanics than on content depth and presentation, suggesting partial loops are easier than polished vertical slices.

Visual iteration matters.

Kimi-K2.6 used 2,998 rendered-screen inspections across 140 tasks, showing that visual feedback is a major debugging signal for game development.

Judge scores are stable on fixed evidence.

Repeated GPT-5.5 judging on fixed gameplay evidence shows small variance compared with the gaps between agents and families.
